# Comprehensive E2E Testing Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add Playwright E2E tests for every page and feature in NestMatch, achieving production-ready test coverage.

**Architecture:** Rewrite the existing 5 test files (broken selectors/URLs) and add ~15 new test files covering all 45+ pages. Use Playwright's `storageState` for efficient auth sharing. Tests are structured as smoke tests (page loads, key elements visible) plus functional tests (forms, navigation, interactions). Chromium-only for speed during development, multi-browser in CI.

**Tech Stack:** Playwright, TypeScript, Chromium (primary), Firefox/WebKit (CI)

---

## Decisions

1. **Run against dev server** (`http://localhost:3000`) — Playwright auto-starts it
2. **Auth strategy:** `auth.setup.ts` project logs in once → saves `storageState` → all authenticated tests reuse it
3. **Chromium-only locally** for speed; multi-browser in CI
4. **Resilient selectors:** Use `getByRole`, `getByText`, `getByPlaceholder`, CSS selectors — NOT data-testid (redesign didn't add them)
5. **Handle empty data:** Tests verify page structure loads, not specific DB content
6. **Test org:** One spec file per feature module
